Project KSB v0.3.11-1
CPU打牌評価 基盤

変更内容
- CPUの完全ランダム打牌を廃止。
- 打牌後の向聴数を最優先して評価。
- 同じ向聴数では受け入れ枚数が多い打牌を優先。
- 表ドラをなるべく保持。
- 対子・刻子候補、隣接牌・両嵌候補を簡易評価。
- 孤立字牌・孤立端牌を比較的切りやすく調整。
- リーチ後のCPUはツモ切りを維持。
- CPUの花牌即抜き仕様を維持。
- デバッグログに「CPU判断：向聴・受入」を表示。

確認方法
1. 通常ツモ・打牌を数巡進める。
2. 南家・西家の手番ごとに「CPU判断」がログへ出ることを確認。
3. AI停止・再開が従来どおり動くことを確認。
4. 花牌を引いたCPUが即抜きすることを確認。
5. リーチ後のCPUがツモ切りすることを確認。

注意
- 今回は牌効率の第一段階です。
- 他家リーチに対する守備、点棒状況、役作り、副露判断は次版以降です。
