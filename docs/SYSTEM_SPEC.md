# Project KSB SYSTEM_SPEC

## v0.3.7B 時点の開発者向け仕様

### 基本構成

Project KSB は HTML / CSS / JavaScript（Vanilla JS）で実装する。
React / TypeScript / Vite / 外部ライブラリは使用しない。

### 山・王牌

- 嶺上牌は合計8枚（4トン）。
- 花牌補充もカン補充も、すべて嶺上牌から取得する。
- 表示上の山残り枚数は通常山だけを数える。
- 嶺上牌は山残り枚数に含めない。

### 流動式ドラ表示牌

- 開始時に表示されるドラ表示牌は初期ドラ1枚のみ。
- カンが成立したときだけ、ドラ表示牌の右側にカンドラ表示牌を1枚追加する。
- ドラ表示牌は最大5枚（初期ドラ1枚 + カンドラ4枚）。
- カンがない局では、カンドラ候補牌は通常ツモ牌として扱う。
- カン時は通常山からカンドラ表示牌1枚、カン裏ドラ牌1枚を確保する。
- そのため、カン1回で表示上の山残り枚数は2枚減る。
- 嶺上ツモ分は残り枚数表示に含めない。
- 内部的には、カン1回で通常山2枚 + 嶺上牌1枚を消費する。
- 新しいカンドラ表示牌・カン裏ドラ牌を確保できない場合はカン不可。

### ドラ計算

#### 表ドラ

- 通常ドラ表示牌とカンドラ表示牌を同じ配列で管理する。
- 表ドラ対象は各ドラ表示牌から決定する。
- 数牌・字牌は通常の次牌をドラ対象にする。

#### 裏ドラ

- 裏ドラ表示牌とカン裏ドラ牌は内部配列で管理する。
- 裏ドラはリーチ成立時のみ有効。
- リーチしていない場合、裏ドラは内部に存在していても得点計算には加算しない。

#### 花牌ドラ（Project KSB 独自仕様）

- 春・夏・秋・冬の花牌は、花牌そのものとしてドラ1を持つ。
- ドラ表示牌またはカンドラ表示牌が花牌だった場合、春夏秋冬の種類に関係なく、すべての花牌が表ドラ対象になる。
- そのため、通常ドラ表示牌が花牌なら、花牌1枚は「花牌ドラ1 + 表ドラ1 = ドラ2」として計算する。
- カンドラ表示牌も花牌なら重複して加算され、花牌1枚でドラ3以上になり得る。
- 裏ドラ表示牌・カン裏ドラ表示牌が花牌の場合も、リーチ時のみ同じルールで加算する。

### デバッグ表示

v0.3.7Bでは以下を確認できる。

- 表ドラ数
- 裏ドラ数
- 合計ドラ数
- ドラ内訳
- 花牌ドラ確認
- 裏ドラ確認（リーチ済み扱い）

## v0.3.8-1 局進行ステートマシン仕様

### 目的

Sprint 3-8-1では、今後の通常ツモ、花牌補充、カン、AI、和了、流局を接続するため、局進行を一元管理するステートマシンを導入する。

### GamePhase

局進行は以下のフェーズで管理する。

- `INIT`: 初期化前
- `DEAL`: 配牌中
- `DRAW`: 現在手番のツモ待ち
- `ACTION`: 現在手番の操作待ち、打牌待ち
- `DISCARD`: 打牌処理
- `NEXT_PLAYER`: 次手番への遷移
- `RYUKYOKU`: 通常山が尽きた流局

### ターン順

手番は `east -> south -> west -> east` の順で循環する。現在手番は `gameState.currentSeat` と `gameState.turnIndex` で管理する。

### 局情報

`gameState` は以下を保持する。

- `roundWind`
- `handNumber`
- `honba`
- `deposit`
- `dealer`
- `currentSeat`
- `turnIndex`
- `phase`
- `turnCount`
- `hands / rivers / melds / flowers / riichi`

### Project KSB 既存仕様との関係

- 嶺上牌は8枚管理を維持する。
- 山残り表示に嶺上牌は含めない。
- カン時はカンドラ表示牌1枚とカン裏ドラ牌1枚を通常山から確保し、表示上の山残りを2枚減らす。
- ドラ表示牌・裏ドラ表示牌が花牌の場合、春夏秋冬すべてを対象にする。
- 花牌自体のドラ、表ドラ、裏ドラ、カンドラ、カン裏ドラは重複加算する。

### 現時点の制限

- CPU思考は暫定ランダム打牌。
- 和了判定、点数計算、鳴き判定、正式な流局処理は未実装。
- 画面表示は自家の手牌・河を中心とし、他家の詳細表示は後続Sprintで拡張する。

## v0.3.8-2 通常ツモ仕様
- 通常ツモは通常山から1枚取得し、表示上の山残りを1枚減らす。
- 自家はツモ後に手牌14枚、打牌後に13枚へ戻る。
- 南家・西家は現在の仮AIとして、自動で1枚ツモしランダムに1枚打牌する。
- 嶺上牌8枚は通常ツモでは消費しない。
- 花牌補充・カン補充は後続Sprintで嶺上牌から処理する。

## v0.3.8-5 カン基盤
- 暗槓・加槓はカン可能条件を満たすと実行できる。
- カン可能条件: カンドラ表示牌を新たに追加できること、カン裏ドラ候補を確保できること、嶺上牌が残っていること。
- カン成立時は通常山からカンドラ表示牌1枚、カン裏ドラ候補1枚を確保するため、表示上の山残りは2枚減る。
- カン後の補充は嶺上牌から行う。嶺上牌は表示上の山残りに含めない。
- 花牌は空気牌であり、手牌利用・打牌・カン対象にはならない。

## v0.3.9-1 シャンテン・テンパイ判定基盤
- 自家手牌について、通常形と七対子のシャンテン数を計算する。
- 花牌は空気牌のため、シャンテン・テンパイ・待ち牌判定の手牌材料から除外する。
- 副露およびカンは通常形シャンテンにおける完成面子として扱う。
- 聴牌時は、全牌種を1枚ずつ仮追加し、和了形になる牌を待ち牌として表示する。
- 本Sprintでは役判定・点数計算・和了宣言処理は実装対象外とする。
- 国士無双系の特殊形は現時点では対象外とする。

## v0.3.9-3 和了基盤仕様
- ロン可能時は進行を一時停止し、ロンボタンを表示する。
- リーチ後は見逃し不可。ロン可能時のキャンセル・見送りはできない。
- 非リーチ時のロン見送りは可能。ただし同巡フリテン等の詳細制御は後続Sprintで実装する。
- ツモ和了可能時はツモ和了ボタンを表示する。
- 花牌は和了・向聴判定の素材から除外する。


## v0.3.9-4 役判定・点数区分仕様

- 食いタンのみでは和了不可。
- 副露タンヤオは、トイトイなど他の役と複合した場合のみ1翻として認める。
- ホンイツは門前3翻、副露2翻で扱う。
- 副露タンヤオ + ホンイツのように食い下がり役と複合する場合、ホンイツは2翻として計算する。
- 点数区分は以下の通り。
  - 5翻: 満貫
  - 6〜7翻: 跳満
  - 8〜10翻: 倍満
  - 11〜13翻: 三倍満
  - 14翻以上: 数え役満
- 13翻ちょうどは数え役満ではなく三倍満。

## 対局終了・順位判定基盤（v0.3.10-5）
- 暫定的に南3局を最終局とする。
- 南3局で親が継続条件を満たした場合は南3局のまま連荘する。
- 南3局で子が和了した場合、または親ノーテンで流局した場合は対局終了とする。
- 対局終了時は点棒の多い順に1位、2位、3位を決定する。
- 同点時の正式ルールは未確定のため、現段階では自家・南家・西家の席順を暫定優先順位とする。
- 返し点、オカ、ウマ、トビ終了、延長戦は今後実装する。
